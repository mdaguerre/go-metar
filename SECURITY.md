# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in go-metar, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Email the maintainer directly or use [GitHub's private vulnerability reporting](https://github.com/mdaguerre/go-metar/security/advisories/new)
3. Include a detailed description of the vulnerability
4. Provide steps to reproduce if possible

## What to Expect

- **Acknowledgment**: You'll receive a response within 48 hours
- **Updates**: We'll keep you informed of our progress
- **Resolution**: We aim to address critical issues within 7 days
- **Credit**: We'll credit you in the release notes (unless you prefer anonymity)

## Scope

This policy covers the go-metar CLI tool and its dependencies. Security concerns related to:

- Command injection vulnerabilities
- Dependency vulnerabilities
- Data handling issues
- API key exposure (if applicable in future versions)

## Out of Scope

- The Aviation Weather Center API (report to aviationweather.gov)
- General bugs that don't have security implications
